# 【管理者向け】AI見積もりシステム セキュリティ管理マニュアル

本システムのアクセス制限（パスワード認証）の管理方法について説明します。

## 1. アクセス認証の仕組み
本システムは、GitHub Pages等の公開環境で安全に運用するため、フロントエンド側に簡易的なパスワードゲートを設けています。

- **認証方式**: クライアントサイド・パスワード認証
- **セッション維持**: ブラウザのタブを閉じるまで有効（sessionStorageを使用）
- **現在のパスワード**: `ccs-bs-expert`

## 2. パスワードの変更方法
パスワードを変更する場合は、以下の手順でソースコードを修正し、GitHubにプッシュしてください。

1. **対象ファイル**: `estimation-ui-app/app.js` を開く。
2. **修正箇所**: 7行目付近の定数を変更する。
   ```javascript
   // 修正前
   const AUTH_PASSWORD = 'ccs-bs-expert';
   
   // 修正後
   const AUTH_PASSWORD = '新しいパスワードを入力'; 
   ```
3. **反映**: 保存後、`git add .` -> `git commit` -> `git push` を実行すると、数分後にデプロイ先にも反映されます。

## 3. 利用者への案内
社内配布用のガイド（`internal_introduction.md`）に、以下の情報を添えて配布してください。

1. **システムURL**: （GitHub Pagesで発行されたURL）
2. **パスワード**: `ccs-bs-expert` （または変更した後の文字列）

## 4. セキュリティ上の留意点
- 本パスワード認証はブラウザ側で判定を行う簡易的なものです。悪意のあるユーザーがソースコード（難読化されていない場合）を解析すればパスワードを特定できる可能性があります。
- そのため、本システムには顧客の個人名などの「個人情報」や「極秘資料」は入力・保存しない運用を徹底してください。
- あくまで「社内向けの見積シミュレーションを便宜上保護する」ための機能として活用してください。
